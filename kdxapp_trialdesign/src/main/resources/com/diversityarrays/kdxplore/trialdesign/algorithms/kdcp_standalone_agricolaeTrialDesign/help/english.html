About

This plugin builds upon the R package: agricolae, providing visualistions and layout data in CSV format

Outputs

The following criteria outlines the files that Plugins in this collection will output, dependening on study type.

Name Format Description
plots.csv CSV Table generated by design method in the Agricolae R pacakge
plots_expdesign_EntryIDGenotype.pdf PDF Illustration of plots colored WRT Genotype/Treatment
plots_fieldnames.csv CSV Illustration of plots identified by treatement in CSV format. Similar to plots_expdesign_EntryIDGenotype
plots_fieldplots.csv CSV Illustration of plots identified by entryID (Unique Identifier) in CSV format. Similar to plots_expdesign_EntryIDGenotype
4KDMan/plots_expdesign.csv CSV Table of Experimental Design Data.
  • Genotype: AKA Treatment. Column identifies allocated Genotype/Treatement.
  • Row: Identifies index in 1-based format
  • Column: Identifies index in 1-based format
  • Block: OPTIONAL. Identifies block/enviroment this unit is allocated to.
  • Replicate: OPTIONAL. Replication number of this unit within its Genotype/Treatment Group.
  • EntryID: Unique identifier
4KDXplore/plots_expdesign.csv CSV Table of Experimental Design Data, with each block seperated into a sepearte file.

This CSV has the following structure. COND tag identifies the field may/may not be present depending on the method:

  • Treatment fields, either single treatment:
    • Treatment: Identifies the Treatment to apply to this entry
  • Or treatment pairs
    • Treatment1: Identifies the first Treatment to apply to this entry
    • Treatment2: Identifies the second Treatment to apply to this entry
  • Row: 1-based position index
  • Column: 1-based position index
  • Block(COND): Environment/block group for this entry
  • Replicate(COND): Replicate number W.R.T. this treatment
